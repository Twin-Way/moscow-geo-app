# Moscow Geo App

Веб-приложение для визуализации и управления географическими объектами на карте Москвы. Приложение позволяет просматривать различные слои (метро, автобусные остановки, пешеходные улицы, районы) и создавать пользовательские точки на карте.

## Технический стек

- **React 19** — UI фреймворк
- **TypeScript ~5.9** — типизация
- **Vite 7.3** — сборщик проекта
- **Deck.GL** — высокопроизводительная визуализация геоданных
- **react-map-gl** — интеграция с Mapbox
- **Zustand** — управление состоянием UI
- **React Query** — кэширование и управление API данными
- **Material-UI (MUI)** — компоненты интерфейса
- **MSW 2.12** — мокирование API запросов
- **ESLint & Prettier** — линтинг и форматирование кода

## Архитектура

Проект организован по принципам **FSD (Feature-Sliced Design)**:

```
src/
├── app/              # точка входа приложения
├── entities/         # бизнес-сущности (модели данных)
│   └── geo-object/
│       ├── model/    # Zustand store
│       ├── api/      # React Query хуки
│       └── lib/      # утилиты и адаптеры
├── widgets/          # переиспользуемые компоненты
│   ├── map/          # компонент карты с модальным окном
│   └── add-object-form/  # форма добавления объектов
├── pages/            # страницы приложения
├── shared/           # общие утилиты, мокирование, стили
│   └── mocks/        # MSW обработчики для API
└── index.css         # глобальные стили
```

## Быстрый запуск (Docker-first)

Этот репозиторий настроен для приоритетного запуска через Docker. Ниже даны команды для быстрого старта в контейнерах.

Dev (hot-reload, рекомендуемый для разработки в контейнере):

```bash
docker compose -f docker-compose.dev.yml up --build
# Открой http://localhost:5175
```

Production (сборка и запуск в nginx):

```bash
docker build -t moscow-geo-app:prod .
docker run -p 8080:80 moscow-geo-app:prod
# Открой http://localhost:8080
```

Файлы, относящиеся к Docker:

- `Dockerfile` — multi-stage production build (сборка через Node, затем раздача `dist` через `nginx`).
- `Dockerfile.dev` — dev-образ с предустановленными зависимостями и командой `npm run dev`.
- `nginx.conf` — конфигурация nginx с history API fallback для SPA.
- `docker-compose.dev.yml` — dev-сервис (билдит `Dockerfile.dev`) для hot-reload.
- `.dockerignore` — исключения для контекста сборки.

Советы:

- На macOS при проблемах с файловым watch сначала попробуйте убрать `CHOKIDAR_USEPOLLING`.
- При запуске через Docker страница может подгружаться заметно дольше, чтобы этого избежать можно использовать альтернативный способ запуска.

## Основной функционал

### 📍 Просмотр геоданных

- Отображение слоёв данных:
  - **Метро** (синие точки)
  - **Автобусы/Трамваи** (красные точки)
  - **МЦД** (зелёные точки)
  - **МЦК** (оранжевые точки)
  - **Пешеходные улицы** (фиолетовые линии)
  - **Районы** (светло-голубые полигоны)

### ➕ Создание пользовательских точек

1. Нажми кнопку **"Добавить точку"** в левом верхнем углу
2. Кликни на карте, чтобы выбрать координаты.
   Если при клике отображается модальное окно с описанием точки, то значит, что в данном месте уже есть какой-то объект. В таком случаем, можно просто нажать **"Закрыть"** в модальном окне и все равно добавить точку.
3. Заполни форму:
   - Название
   - Тип объекта
   - Описание
4. Нажми **"Сохранить"**

Пользовательские точки:

- Отображаются **серым цветом**
- Сохраняются в **localStorage** браузера
- Persists при перезагрузке страницы

### 🔍 Просмотр деталей объекта

Кликни на любой объект на карте, чтобы открыть модальное окно с подробной информацией.

## Работа с API

### API обработчики

API запросы мокируются через **MSW** (Mock Service Worker). Обработчики находятся в [src/shared/mocks/handlers.ts](src/shared/mocks/handlers.ts).

Доступные эндпоинты:

- `GET /api/geo` — получить все геоданные (GeoJSON)
- `POST /api/geo` — создать новую пользовательскую точку

### Работа с моками

Service Worker файл `public/mockServiceWorker.js` автоматически генерируется пакетом `msw`. **Не редактируй этот файл вручную**.

Чтобы отключить моки (переключиться на реальный API):

1. Открой [src/main.tsx](src/main.tsx)
2. Закомментируй строку: `// await startMSW();`

## Управление состоянием

### Zustand Store (entities/geo-object/model/store.ts)

```typescript
const { select, setAddMode, setTempCoords } = useUIStore();
```

- `select(feature)` — выбрать объект для просмотра
- `setAddMode(bool)` — переключить режим добавления точки
- `setTempCoords([lon, lat])` — сохранить временные координаты

### React Query (entities/geo-object/api/useGeoObjects.ts)

```typescript
const { data, isLoading } = useGeoObjects();
// data — GeoJSON FeatureCollection
```
